## 1. Context & Objective

The Providers Silver layer solves a **critical referential integrity issue**: claims reference providers that don't exist in the Bronze providers dimension.

This layer transforms **inconsistent provider data** into a **complete, validated provider dimension** that:

- 📋 Ensures every claim has a valid provider  
- ✅ Enforces provider FK integrity (100%)  
- 🔗 Preserves fraud risk attributes  
- 📊 Adds lineage & audit trails  
- 🔒 Enables provider-level analytics  

**Enterprise standards applied:**
- Schema enforcement  
- Primary key validation  
- FK integrity guarantee  
- Fraud label preservation  
- Audit-trail metadata  
- Deduplication  
- Metrics logging

## 2. Bronze vs Silver Providers

| Area | Bronze Issues | Silver Solution |
|------|---------------|-----------------|
| **Provider Coverage** | Claims ref providers NOT in Bronze | Union of Bronze + claims-derived |
| **FK Guarantee** | Claims → Providers = 0% match | 100% referential integrity |
| **Schema** | Inconsistent types | Enforced schema (ID, Fraud_Flag) |
| **Fraud Labels** | Only partial labels | Original labels preserved + defaults |
| **Label Lineage** | No audit trail | Fraud_Label_Source field added |
| **Duplicates** | Possible | Deduplicated by Provider_ID |
| **Quarantine** | N/A | Invalid IDs isolated |

**Result:** Complete provider dimension enabling provider-level fraud, cost, and performance analytics.

## 3. Transformations: Bronze → Silver

### Read Bronze Providers & Claims

- Bronze Providers: ~5K rows (Kaggle fraud dataset)  
- Bronze Claims: Contains 5000+ unique Provider_IDs  
- **Problem:** Claims providers ≠ Bronze providers (data mismatch)  

### Union: Bronze + Claims-Derived

```python
# Step 1: Get provider IDs from Bronze
bronze_providers = read_bronze("providers")

# Step 2: Extract unique providers from claims
claims_provider_ids = distinct(
    read_bronze("claims").select("Provider_ID")
)

# Step 3: Union — treat claims as system of record
all_providers = bronze_providers.union(claims_provider_ids)
```

**Result:** Includes both original Bronze providers AND any provider appearing in claims.

### Schema Enforcement

Cast columns to enterprise schema:
```
Provider_ID (string),
Fraud_Flag (int, 0/1),
Fraud_Label_Source (string)
```

### PK Validation

`Provider_ID` must not be NULL → otherwise quarantined.

### Deduplication

Keep unique Provider_IDs:
```
drop_dupes_keep_latest(["Provider_ID"])
```

### Fraud Label Logic

For each provider:
- **If in Bronze Providers:** Keep original `Fraud_Flag` (0 or 1)  
- **If only in Claims:** Set `Fraud_Flag = 0` (no fraud evidence)  
- **Add field:** `Fraud_Label_Source` ∈ {`bronze_label`, `inferred_from_claims`}  

**Business rationale:** Absence of fraud evidence ≠ fraud, so default is safe (0).

### Write to Silver

- **Delta path:** `silverdata/providers`  
- **Table:** `bupa_silver.providers`  
- **Schema-aware overwrite** ensures stability  

### Metrics Logging

Tracked in `silverdata/_metrics`:
- `rowcount_providers_silver`  
- `distinct_provider_ids`  
- `fraud_label_coverage_%`  
- `label_source_distribution` (bronze vs claims)

## 5. The FK Fix: Claims → Providers

### The Problem (Bronze)

| Metric | Value |
|--------|-------|
| Claims with valid Provider_FK | 0% ❌ |
| Orphan claims (no provider match) | 100% ❌ |
| Business impact | Cannot do provider analytics |

### The Solution (Silver)

| Metric | Value |
|--------|-------|
| Claims with valid Provider_FK | 100% ✅ |
| Orphan claims | 0% ✅ |
| Business impact | Provider analytics now reliable |

### How It Works

1. Extract all unique Provider_IDs from Bronze Claims  
2. Add them to the Silver Providers table  
3. Mark them with `Fraud_Label_Source = "inferred_from_claims"`  
4. Result: Every claim now has a matching provider in Silver  

**Business value:**
- ✅ Enables Claims → Providers joins in dashboards  
- ✅ Allows provider-level fraud scoring  
- ✅ Supports provider performance analysis  
- ✅ Powers provider risk segmentation
